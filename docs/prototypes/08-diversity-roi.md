# 8. 多様性ROI版

## コンセプト
効率だけを正解にせず、満足、家族時間、快適さ、新しい発見など、その日の価値観に応じて選べる統合案です。ROIとQOLを別の指標として扱います。

## 検証したいこと
- 3案をランキングではなく、異なる価値の選択肢として見せられるか
- 利用者が価値の重みを変えることで、提案への納得感が高まるか
- 個人への価値と東京への副次的な価値を混同せずに伝えられるか

## 主なフロー
1. 今日の過ごし方と条件を選ぶ
2. 必要に応じて大切にしたい価値を調整する
3. スマート、ゆとり、発見の3案を確認する
4. 詳細比較後に一案を選ぶ
5. 振り返りとMy ROIで選択傾向を確認する

## 表示する価値
各案は時間・費用・混雑だけでなく、家族時間、快適さ、発見度を異なる重点で提示します。総合ROIは詳細内の補助指標であり、唯一の正解にはしません。

## モック範囲
4つの気分シナリオと固定候補データを使用します。選択・振り返りはローカルストレージへ保存します。
