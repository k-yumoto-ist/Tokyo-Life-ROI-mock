# 4. AIバトル型

## コンセプト
異なる強みを持つ3プランを対戦形式で見比べ、最後にAIが総合判定を公開する案です。ユーザー自身の「推し」とAI判定を分け、決定権は利用者に残します。

## 検証したいこと
- 比較体験が退屈にならず、選択の理由を理解できるか
- 総合ROIを最後に公開することで、数値に引きずられずに特徴を見比べられるか
- AIの判定と利用者の価値観が異なっても納得できるか

## 主なフロー
1. 条件を入力して3プランを生成する
2. 時間、節約、混雑回避、満足度、家族適性を比較する
3. 推しのプランを選ぶ
4. AI判定でROIと勝者を確認する
5. 最後は利用者がプランを決定する

## 表示する価値
バトル中は総合ROIや順位を表示しません。AI判定後にのみ最終ROIと比較結果を表示します。

## モック範囲
近場速攻型、節約じっくり型、満足度全振り型の固定データで動作します。
