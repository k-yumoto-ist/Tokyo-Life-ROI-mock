# 2. 条件入力型

## コンセプト
目的地や利用条件を明示し、複数候補を比較して決める案です。目的地が決まっている移動と、目的から候補を探す行動の両方を扱います。

## 検証したいこと
- 条件入力から提案までの納得感
- 時間・費用・混雑・同行者を比較材料として使えるか
- 個人設定が提案理由に反映されていると感じられるか

## 主なフロー
1. 行き方または過ごし方を選ぶ
2. 出発地、目的地、時間、予算、同行者などを入力する
3. 3案の候補を比較する
4. 一案を選び、行動・振り返りへ進む

## 表示する価値
所要時間、費用、徒歩、混雑、満足度予測、パーソナルROIを候補ごとに表示します。

## モック範囲
入力内容に応じて固定の候補群を切り替えます。検索候補や交通情報はデモデータです。
